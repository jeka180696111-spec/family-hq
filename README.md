## 🚀 Як підключити в гітхаб (окрема гілка)

```bash
# 1. Клонуй свій репозиторій локально (якщо ще ні)
git clone https://github.com/<твій_юзер>/<репозиторій>.git
cd <репозиторій>

# 2. Створи нову гілку
git checkout -b v3-redesign

# 3. Скопіюй ВСІ файли з папки v3 у корінь проекту
#    (замінюючи старі)
# Або через rsync:
# rsync -av --delete /path/to/v3/ ./

# 4. Закомить і запуш
git add .
git commit -m "v3: модульна архітектура + redesign + єдина сторінка кошельків"
git push origin v3-redesign
```

### 4. Vercel preview deploy

Vercel автоматично створить **preview deployment** для гілки `v3-redesign`. URL буде типу:
```
https://<твій_проект>-v3-redesign-<юзер>.vercel.app
```

Старий `main` залишається працювати на основному URL.

### 5. Коли впевнишся що все OK — мерджимо:

```bash
git checkout main
git merge v3-redesign
git push origin main
```

## ⚠️ ВАЖЛИВО перед першим запуском

### 1. URL скрипта (вже в коді)
В `js/config.js` рядок 7:
```js
SCRIPT_URL: 'https://script.google.com/macros/s/AKfycbyJcoxq4M.../exec'
```
Якщо твій URL інший — заміни.

### 2. Очисти localStorage
Старі дані з v2 можуть конфліктувати. У браузері (F12 → Console):
```js
localStorage.clear();
location.reload();
```

### 3. Code.gs залишається той самий
Бекенд не міняється — v3 повністю сумісний з тим самим `Code.gs v2.2` що ти вже залив.

## 🐛 Якщо щось не працює

### "Module not found"
Vercel сам розрулює — це працює "з коробки" бо у нас `type="module"` і відносні шляхи. Локально треба запускати через `npx serve` (не `file://`).

### "Cannot read properties of null"
Скоріш за все, якийсь модуль ще не завантажився. Спробуй жорстке перезавантаження (Ctrl+Shift+R).

### Google Sign-In не показується
Перевір `GOOGLE_CLIENT_ID` у `js/config.js`. Має бути валідний OAuth Client ID. Якщо змінив домен (Vercel дав новий URL) — додай його в Google Cloud Console → OAuth credentials → Authorized JavaScript origins.
